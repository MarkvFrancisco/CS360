## Briefly summarize the requirements and goals of the app you developed. What user needs was this app designed to address?

### The requirements of the Event Tracking Mobile Application was to effectively implement a login screen utilizing a database for creating and storing user information (username and password), implement a database shell that is displayed on a grid which utilizing all CRUD operations when it comes to event management, and to allow users to select whether they would like to receive SMS notifcations when an event is within one week of the current day. The app should function properly when the user opts out of receiving SMS notifications. 

### The goals of the Event Tracking Moile Application is to effectively address all software requirements and user needs through implementing aspects to software such as accessiblity and  layout of the user interface. The application must also comply with industry-standard best practices so as to make code readable, maintainable, and scaleable.  

## What screens and features were necessary to support user needs and produce a user-centered UI for the app? How did your UI designs keep users in mind? Why were your designs successful?

### One of the screens necessary to support user needs that produce a user-centered UI for the mobile application was the simplistic login screen that includes the 'Create Account' button to save the username and password to a datbase, as well as the 'Login' button to check whether the entered username and password matches any of the items stored within the database. Features necessary that support user needs was by including SMS notifications allowing users to select when they want to be notified when a particular event is approaching the one week mark. This allows users to always stay updated on upcoming events. 

## How did you approach the process of coding your app? What techniques or strategies did you use? How could those techniques or strategies be applied in the future?

### I approached the process of coding my mobile application through gathering all necessary technical requirements, building the application, then making final adjustments. I built the app by organizing the technical requirements into individual components such as the frontend UI, java logic, database logic, and SMS logic. The strategies that I used was by implementing a user-centric UI to fulfill user requirements in a streamline fashion as well as following an Agile approach to software development – at least up to the deployment stage of the Agile process. 

## How did you test to ensure your code was functional? Why is this process important, and what did it reveal?

### I tested my code by constantly running the application on an Android Device Simulator so that I know when and how the application could crash and make the necessary adjustments to the code. This process is important because it allows developers to catch runtime errors early in the development process to prevent large codebase refactoring. Regularly testing code revealed many of my blind spots in knowledge in database logic and various libraries which prompted me to research how they work and how to effectively implement them into the application.
